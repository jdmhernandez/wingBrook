Long-term Goal, Sprint Questions & HMWs

Long-term Goal

Create an intuitive membership application system that helps Focus Photography Club sustainably grow while collecting meaningful member data and generating revenue.

Sprint Questions

How might we balance collecting comprehensive member information without overwhelming applicants?

How might we design a membership process that accommodates users with varying technical abilities?

How might we create a platform that scales with the club's growing needs?

How might we ensure the portfolio submission process is straightforward?

How Might We (HMWs)

  • HMW make the experience feel approachable for beginners while still appealing to advanced photographers?
  • HMW design a mobile-friendly form that handles image uploads intuitively?
  • HMW communicate membership benefits during the application process?
  • HMW clearly explain the approval process to applicants?
  • HMW design an intuitive portfolio upload that enforces file type and size restrictions?

Handwritten Notes

Key Observations

  • Club's transition from casual meetup to structured organization requires formalized membership system
  • Current limitations of Meetup.com platform: no direct contact, limited member information, no payment system
  • Diverse user group with varying technical abilities (all ages, experience levels)
  • Need for financial sustainability through $40 annual membership fees
  • Challenge: Balance comprehensive data collection with simplicity to prevent form abandonment
  • Mobile responsiveness critical for accessibility
  • Portfolio submission must handle various file types while enforcing size limits (10MB)
  • Data needs: demographic info, skill levels, equipment types, portfolio samples, interests
  • Must communicate clear value proposition to justify membership fee
  • Need clear path for application review and approval process

User Needs

  • Simple, intuitive application process
  • Clear explanation of membership benefits
  • Ability to showcase photography skills/interests
  • Transparency about approval process
  • Mobile-friendly interface
  • Accessibility for users of all technical abilities
  • Simple file upload process

Club Needs

  • Comprehensive member information collection
  • Ability to segment members by experience level
  • Portfolio samples to assess skill level
  • Understanding of members' interests for event planning
  • Sustainable revenue through membership fees
  • Agreement to club terms and code of conduct

Possible Solutions

Solution 1: Wizard-Style Application

üìã

A multi-step linear flow breaking the application into logical sections with a progress indicator showing completion status. Each screen focuses on one component of the application, making it less overwhelming and more mobile-friendly.

Pros: Clean, focused interface; reduces cognitive load; mobile-friendly; clear progress tracking

Cons: Multiple steps might lead to higher abandonment rates; harder to review entire application at once

Solution 2: Single-Page Application

üìÑ

All form fields visible on one scrollable page with collapsible sections for organization. A floating "Apply" button remains visible while scrolling, and the portfolio uploader uses drag-and-drop functionality.

Pros: Users can see all requirements at once; easier to navigate back and forth; single submission

Cons: May appear overwhelming; longer scroll on mobile; form might feel daunting

Solution 3: Conversational Interface

üí¨

A chatbot-style interface that asks questions sequentially in a conversational tone. Includes built-in help and examples for each question and an interactive portfolio upload with thumbnail previews.

Pros: Friendly, approachable feel; mirrors club's casual origins; can provide contextual help

Cons: Less efficient for advanced users; may feel gimmicky; harder to implement complex functionality

Storyboard of Best Solution

Based on our evaluation, the Wizard-Style Application provides the best balance of simplicity, guidance, and comprehensive data collection. Here's a simplified storyboard illustrating the key steps in the user flow:

1
1 User discovers and explores membership benefits

User finds the club website, clicks on "Join Us", and reviews membership benefits and the $40 annual fee on a dedicated landing page.

2
2 Personal information entry

User enters basic personal information (name, DOB, contact details) in a simple form with clear validation feedback.

3
3 Photography experience and equipment

User selects experience level (beginner, intermediate, advanced) and equipment used through visual selector cards and checkboxes.

4
4 Portfolio and interests

User uploads portfolio photos (up to 5) with an intuitive drag-drop interface and enters photography interests using a tag-based system.

5
5 Reason for joining and terms agreement

User enters reason for joining (with word counter) and agrees to club terms and code of conduct with clear links to full documents.

6
6 Review, submit, and next steps

User reviews all information, submits application, receives confirmation with next steps explanation and timeline for approval.
